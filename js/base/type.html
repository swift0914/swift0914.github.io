# JS 判断类型方式

# 1、typeof

适用场景

typeof 1; //number
typeof "1"; //string
typeof Symbol(); // symbol
typeof undefined; //undefined
typeof false; //boolean

typeof function() {}; // function

# 2、instanceof

instanceof 操作符可以帮助我们判断引用类型具体是什么类型的对象

[] instanceof Array; // true
new Date() instanceof Date; // true
new RegExp() instanceof RegExp; // true
1
2
3

[] instanceof Array 实际上是判断 Array.prototype 是否在[]的原型链上,所以，使用 instanceof 来检测数据类型，不会很准确，这不是它设计的初衷

[] instanceof Object // true
function(){}  instanceof Object // true
1
2

注意

另外，使用 instanceof 也不能检测基本数据类型，所以 instanceof 并不是一个很好的选择。

# 3、toString

我们可以直接调用 Object 原型上未被覆盖的 toString()方法，使用 call 来改变 this 指向来达到我们想要的效果。

Object.prototype.toString.call(1) [object Number]

Object.prototype.toString.call(null) [object Null]
Object.prototype.toString.call(undefined) [object Undefined]
Object.prototype.toString.call(Symbol()) [object Symbol]
Object.prototype.toString.call({}) [object Object]
Object.prototype.toString.call(function(){}) [object Function]
Object.prototype.toString.call([]) [object Array]

# 4、仿 jquery

let class2type = {};

// 生成class2type映射
"Boolean Number String Function Array Date RegExp Object Error Null Undefined"
    .split(" ")
    .map((item) => {
        class2type["[object " + item + "]"] = item.toLowerCase();
    });

// typeof 除了引用类型都可以正确判断
let type = (obj) => {
    // IE6 undefined和null Object.prototype.toString.call()都返回 [object Object] 需要做兼容
    if (obj == null) return obj + "";
    return typeof obj === "object" || typeof obj === "function"
        ? class2type[Object.prototype.toString.call(obj)] || "object"
        : typeof obj;
};

console.log(type({})); //object
console.log(type(null)); // null
