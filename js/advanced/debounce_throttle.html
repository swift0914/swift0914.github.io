# 防抖、节流函数

# 函数防抖(debounce)

防抖和节流是都预防函数的对此调用，区别在于假设用户重复触发函数，每次触发的时间间隔小于预设时间，防抖函数只会执行一次，而节流函数会已一定的时间间隔调用一次函数。

TIP

在事件触发 N 时间后在触发回调，如在期间再次触发则重新计算。

let debounce = function(fun, wait) {
    let timer = null,
        context = this,
        args = arguments;
    return function() {
        if (timer) clearTimeout(timer);
        timer = setTimeout(() => {
            func.apply(context, args);
        }, wait);
    };
};

# 函数节流(throttle)

TIP

在规定时间内只触发一次函数。只有大于规定周期才会执行第二次

let throttle = function(fun, wait) {
    let prev = 0,
        context = this,
        args = arguments;
    return function() {
        let now = Date.now();
        if (now - prev >= wait) {
            fun.apply(context, args);
            prev = Date.now();
        }
    };
};

# 总结

  • 防抖和节流都是防止事件频繁触发，但是原理不一样。
  • 防抖是某一段时间内只执行一次，而节流是间隔时间执行。

# 使用场景

  • 防抖

    • 输入框只有在用停止输入时采取进行ajax请求。
    • window触发resize的时候，不断改变浏览器的大小，使用防抖只触发最后一次。

  • 节流

    • 监听滚动事件，例如上拉加载时使用节流，有效的节约请求次数。
    • 鼠标、touch事件。
